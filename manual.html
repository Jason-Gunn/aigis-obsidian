AIGIS for Obsidian

AIGIS for Obsidian is a focused governance plugin for teams that want local Markdown records for AI systems, prompts, policies, workflows, skills, and incidents.

Overview

This plugin is a local governance layer for Obsidian. It does not call model providers, execute prompts, or provide server-side analytics. It creates structured notes, keeps a dashboard up to date, and appends simple audit entries for bootstrap actions and note creation.

Structured notes

Each module creates Markdown notes with predictable frontmatter and starter sections.

Console panel

The side panel shows module counts and quick actions for note creation, bootstrap, dashboard, and audit log access.

Lightweight audit log

The plugin records bootstrap events and note creation in a plain Markdown log.

Vault Structure

The default managed root is AIGIS/. Bootstrap creates any missing folders and files, refreshes Dashboard.md, and appends a vault.bootstrap entry to the audit log.

AIGIS/ |-- Dashboard.md |-- User Guide.md |-- Inventory/ |-- Prompts/ |-- Policies/ |-- Workflows/ |-- Skills/ |-- Incidents/ `-- Audit/ `-- Audit Log.md

Installation

Open this repo as a vault

  1. Open the AIGIS-Obsidian folder as a vault in Obsidian.
  2. Go to Settings -> Community Plugins.
  3. Enable AI Governance and Infrastructure Suite for Obsidian.
  4. Run AIGIS: Bootstrap governance vault.

Add the plugin to an existing vault

  1. Create <your-vault>/.obsidian/plugins/aigis-governance/.
  2. Copy main.js, manifest.json, and styles.css into that folder.
  3. Reload community plugins and enable AI Governance and Infrastructure Suite for Obsidian.
  4. Optionally change Root folder in Settings -> AIGIS Governance.
  5. Run AIGIS: Bootstrap governance vault.

Development

cd ".obsidian/plugins/aigis-governance"
npm install
npm run dev

Settings

Auto-open consoleonOpen the console on startup.

Console visibility is controlled in settings only. The console itself does not include hide or show toggles.

Use the arrow buttons in settings to reorder cards. Console appearance settings control card background, borders, spacing, title styling, count styling, description styling, and the create button colors.

Security Notes

Filenames are slugified from note titles. Audit entries are normalized to a single line to reduce log-forging risk. The audit log is plain Markdown and should be treated as advisory, not tamper-proof. Settings-backed paths are sanitized before the plugin uses them.
